# 持续改进（负反馈机制）

这套机制让系统从每次"出错"中学习，类似免疫系统——遇到一次问题，产生抗体，永不复发。

## 触发条件

当以下任一信号出现时，启动反馈捕获：

### 🔴 立即触发（对话中实时响应）

| 信号 | 示例 | 动作 |
|------|------|------|
| 显式纠正 | "不对"、"不要这样"、"以后别..." | 立即起草规则更新 |
| 重复问题 | 同类错误出现第 2 次 | 标记为高频问题，升级优先级 |
| 新偏好表达 | "我习惯用..."、"一般我都是..." | 记录为个人偏好 |
| 确认的有效做法 | "对，就这个思路"、"这个方案好" | 记录为已验证模式 |

### 🟡 延迟触发（会话结束复盘时）

| 信号 | 示例 | 动作 |
|------|------|------|
| 多次修改同一文件 | 一个功能来回改了 3 次以上 | 分析根因，看是需求不清还是规则缺失 |
| 用户表达挫败 | "怎么又..."、"还是不对" | 优先处理 |
| 新工作流出现 | 做了一件之前没做过的事 | 考虑抽象为技能 |

## 反馈分类与存储

```
.claude/learned/
├── feedback/       # 纠正性规则（"不要做X"）
├── patterns/        # 已验证的有效模式（"做X时用Y方法"）
├── preferences/     # 个人偏好（"我喜欢用..."）
```

### feedback/ — 纠正性规则
```markdown
---
type: feedback
trigger: {触发信号}
severity: must | should | may
date: {日期}
---
{规则内容}

**为什么**：{原因}

**适用场景**：{什么时候要注意这个规则}
```

### patterns/ — 已验证模式
```markdown
---
type: pattern
domain: {领域：如 python/react/git}
verified: {验证次数}
date: {日期}
---
{模式描述和示例代码}
```

### preferences/ — 个人偏好
```markdown
---
type: preference
category: {工具/风格/流程}
date: {日期}
---
{偏好描述}
```

## 执行流程

```
1. 检测到触发信号
   ↓
2. AI 起草规则更新（一句话描述）
   ↓
3. 询问用户确认："要不要把这条规则记下来？[Y/N]"
   ↓
4. 用户确认 → 写入对应文件 + 更新 LEARNED.md 索引
   用户拒绝 → 不写入，但记住本次对话不再问同样的问题
   ↓
5. 下次对话自动加载 LEARNED.md，所有已学规则生效
```

## 原则

- **先问再写**：绝不未经用户确认就修改规则文件
- **一条规则一件事**：不把多个纠正打包成一条
- **写清楚为什么**：每条规则必须解释原因，方便日后判断是否还适用
- **可过期**：如果用户说"这个规则不再适用了"，立即删除
- **不重复**：写入前检查是否已有类似规则，有则合并而非新建
