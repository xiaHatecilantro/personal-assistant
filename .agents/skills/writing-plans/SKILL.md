# 实现计划

将开发任务分解为小步可验证的计划。

## 计划格式

每步必须包含：
- **文件**：精确路径
- **改动**：一句话描述做什么
- **验证**：如何确认完成

```markdown
## 计划：{任务名}

### Step 1: {步骤名}（预计 2-5 分钟）
- 文件：`src/user/auth.ts`
- 改动：添加 validateToken 函数，检查 token 格式和过期时间
- 验证：运行 `npm test -- auth.test.ts`，新增 3 个测试用例通过

### Step 2: ...
```

## 拆分原则

- 每步改动不超过 50 行，不超过 2 个文件
- 每步有独立的验证方式（测试/运行/肉眼检查）
- 步骤间尽量解耦，失败时不影响已完成的步骤
- 如果一步太大拆不动，说明需求还需要澄清

## 适用场景
- 涉及 3 个以上文件的新功能
- 重构任务
- 任何拿不准"从哪开始"的任务

## 不适用
- 单行修复（直接改）
- 纯查询问题（直接答）
