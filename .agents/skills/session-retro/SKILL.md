# 会话复盘

在每次有实质产出的会话结束时（或用户说"复盘"时），执行简短复盘。

## 复盘维度

### 1. 纠正记录
- 这次会话中用户纠正了我几次？
- 每次纠正的根因是什么？（规则缺失 / 需求理解错误 / 技术细节错误）
- 哪些纠正应该固化为永久规则？

### 2. 效率问题
- 有没有来来回回改同一个地方超过 2 次？
- 有没有因为缺少信息走弯路？
- 有没有做了又撤销的无效工作？

### 3. 新发现
- 用户有没有表现出新的编码习惯/偏好？
- 有没有出现新的工作模式（如"每次改完 API 都要同步更新文档"）？
- 有没有值得抽象为技能的重复性工作？

## 复盘输出

每轮复盘输出一个简短摘要：

```
## 会话复盘

### 待记住的规则（建议写入）
1. {规则描述} — 因为 {触发原因}

### 已解决（不需要写入）
- {问题} → {本次已解决，不需要额外规则}

### 观察（暂不写入，继续观察）
- {模式} — 再出现一次就记
```

## 操作

复盘完成后：
1. 把"待记住的规则"逐条请用户确认是否写入
2. 用户确认的写入对应 learned/ 目录
3. 用户拒绝的丢弃
4. 更新 LEARNED.md 索引
